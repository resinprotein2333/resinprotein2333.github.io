使用Fluid主题美化Hexo博客

本文最后更新于 2024年1月25日 10:24

导语

上一次我们已经使用 Hexo 博客框架生成了了一个基本的博客网页并且搭建了可以用于存放图片的图床,并且在 GitHub 上也进行了第一次提交.可是 Hexo 默认的主题未免有些单调,这一次就使用 Fluid 来配置一下博客的主题使其变得更美观一些.

如果想要自行获取主题可以从官方的主题配置中选择,点击 这里 即可跳转

这里就以我自己使用的 Fluid 主题为例并进行配置.

配置过程

如果有不详尽的地方可以自行查看 官方文档 ,官方的文档写的很详细,可以按需进行复制粘贴,本文写的时候也对官方的文档进行了一些借鉴,并且将一些官方文档中没有提到的一些功能写出来,希望能帮到你.

安装主题

对于安装主题现在有两种方式分别是使用 npm 命令进行安装,还有一种便在 GitHub 下载源文件,这里对两种方式分别进行介绍

两种方法各有好处,下载源文件可以方便对主题文件本身进行更改,使用 npm 进行安装 themes 目录下就不会有任何主题文件了,好在 Fluid 主题提供了一套易于更新的升级方式,所有需要的文件可以在主题配置文件里更改,所以还是更加推荐第一种,由读者自行按自身需求选择安装方式,关于主题配置文件稍后会提到,在此不过多赘述.

因为配置主题是一件麻烦事,如果想将配置全部写出来会很麻烦,篇幅限制,这里只进行一些简单的配置

Fluid 官方给出了一种配置主题的方式,使得想要配置主题的样式不需要再去修改 _config.yml 有效防止了不小心乱删配置文件内容导致的不可挽回的后果,操作方法如下:

在博客的根目录下新建 _config.fluid.yml 文件即可

这样的话对主题的配置就和默认配置文件分开了,以后将对于主题的配置直接写在 _config.fluid.yml 文件中即可,本文也将以此为基础,后续对主题的配置都将配置在此文件中.

PS: 但是像 Lanuage 或是 Deploy 这些选项仍需要在 _config.yml 中进行配置,也只有在此处配置才能生效.
