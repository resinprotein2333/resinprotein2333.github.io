使用Github仓库 + Picgo开源图床工具托管博客图片

本文最后更新于 2024年1月24日 08:50

导语

在写博客的时候我们通常会需要导入一些图片,但是直接将过多的图片存放于本地的话时间长了又会拖慢博客站的访问速度,对访问的读者造成不便.

同时,如果想要在其他平台进行引用自己的图片的话又很不方便,需要重新进行上传,很不方便,那么在这个时候,就需要用到图床了.

准备工作

在开始之前需要你拥有以下东西:

  • Github 账号
  • Picgo 开源图床软件

❗PS: 如果还没有 Github 账号的可以注册一个.如果不想使用bata版本的软件的话,下载 Picgo 的时候需要注意是不是 Bata 版本

搭建过程

在拥有Github账号之后,先创建一个新的仓库

名字这里可以填你的图床仓库的名字,下面选择 public

如果你想的话,还可以添加一个 README 文件来对你的仓库进行说明,让更多人可以直观的了解这个仓库.

仓库这个样子就说明创建成功了

对GitHub和Picgo进行配置

到了这一步其实图床已经搭建的差不多了,但是直接通过GitHub上传图片未免有些麻烦,所以就需要用到刚刚准备好的 Picgo 图片上传软件来辅助完成图片上传.至于安装的部分便不再细讲,根据官网的引导做就行.

安装完毕之后,他会默认收缩在隐藏的图标中,右键显示主界面即可打开后是这个样子:

之后所需要做的便是对 GitHub 和 Picgo 进行分别的设置,我们先从GitHub的设置开始

配置GitHub

  1. 我们首先需要向 GitHub 申请一个 token.
▶
关于GitHub Personal Access Token

GitHub Personal Access Token是一个用于身份验证的令牌(token)，它可以代替使用用户名和密码进行身份验证。 GitHub Personal Access Token是由GitHub颁发的，只有经过授权的用户才能使用。

依次打开 Settings -> Developer settings -> Personal access tokens

最后点击 generate new token (classic)

  1. 填写并勾选相关信息

填写博客用途之后并勾选 repo 选项 然后点击 Genetate token 即可

出现以下提示栏则说明 token 创建成功

❗PS: 请将 token 转移至安全的地方保存, token 只会显示这一次,丢失或遗忘则需要重新创建.

配置Picgo

打开 Picgo 并转到 图床设置 -> GitHub图床

然后点击确定 如果想要将图床设为默认,则可以点击 设为默认图床

根据所需的信息填写即可,下面两个不重要的选项可以留空

之后便可以在图片上传区进行上传,记得选择自己想要的链接样式,拖拽或是从文件夹选取图片都是可以的

此外,如果想要将图片整理到文件夹中可能会有点不方便,需要重新到设置选项中设置文件夹的信息,这一点还是有点不方便,希望以后的版本可以改进.

想要在博客中引用图片的话直接在相册中,点击下方小图标就能直接把链接拷贝到剪切板.

Hope You Can Enjoy And Have A Nice Day!
